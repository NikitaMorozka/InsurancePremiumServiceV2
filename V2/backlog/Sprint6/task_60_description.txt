**Зачем это было сделано? Чему научился в процессе выполнения задания?**
Логирование - это процесс записи информации о работе программы в специальные файлы или другие хранилища.
Эти данные помогают вылавливать ошибки и выявлять поведение программы.

Зачем логгировать ?
1) Выявление ошибок. Помогает быстро устранять ошибке в кодовой базе
2) Анализ производительности. Помогает выявить узкие места в приложении и провести тем самым оптимизацию
3) Мониторинг: Обеспечивает контроль за состоянием системы в реальном времени.

Как логгировать в Java ?

В Java существуют различные библиотеки для логирования, каждая со своими особенностями. Они обычно включают:
Логгер (Logger): Отвечает за создание записей в журнале.
Appender: Определяет, куда будут записываться логи (например, в файл, консоль или базу данных).
Layout: Форматирует записи лога перед их выводом.

Логирование в Java можно настроить с помощью различных библиотек. Одна из самых популярных — SLF4J (Simple Logging Facade for Java).
Она предоставляет унифицированный интерфейс для работы с различными системами логирования, такими как Log4j, Logback и java.util.logging.
